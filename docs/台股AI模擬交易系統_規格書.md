## 1. 專案概述

### 1.1 目的

本系統為個人使用的台股模擬交易練習工具，整合即時行情資料、技術指標分析與 AI 買賣建議，協助使用者在無風險的虛擬資金環境下練習波段交易策略，並透過歷史回測與績效追蹤驗證策略效果。

### 1.2 使用者

- **對象**：個人投資者（單一使用者）
- **技術背景**：非專業開發者，無機器學習基礎
- **交易風格**：以波段交易為主（持股數天至數週），架構預留當沖擴充空間

### 1.3 核心目標

- 提供即時台股行情監控（最多 5 檔自選股）
- 透過 AI 分析技術指標，給出買賣時機建議與理由說明
- 模擬下單、記錄損益，累積交易紀錄
- 支援歷史回測，驗證 AI 策略的實際績效
- 有訊號時自動發送 Gmail 通知
- AI 引擎可在介面上自由切換，不綁定單一服務

## 2. 技術架構

### 2.1 技術選型

| 元件 | 技術 / 服務 | 說明 |
|------|------------|------|
| 程式語言 | Python 3.10+ | 主要開發語言 |
| 後端框架 | FastAPI | 提供 REST API 與 WebSocket，傳遞即時行情給前端 |
| 使用者介面 | HTML + JavaScript | 網頁看盤介面，無需額外框架 |

## 3. 功能規格

### 3.1 自選股管理

- 最多同時設定 5 檔股票代號（受 Fugle 免費版 WebSocket 上限限制）
- 支援上市、上櫃個股及 ETF
- 自選清單儲存於 SQLite，重啟後保留設定
- 可隨時新增、刪除自選股

### 3.2 即時行情

- 透過 Fugle WebSocket 持續接收即時報價
- 每檔股票顯示：現價、漲跌金額、漲跌幅（%）、成交量、最高 / 最低價
- 漲跌以顏色區分（紅色 = 上漲，綠色 = 下跌）

### 3.3 技術指標計算

以下指標由本地 pandas-ta 計算，**不消耗任何 API 費用**：

| 指標 | 用途 | 預設參數 |
|------|------|---------|
| MA5 / MA20 | 短中期趨勢方向，判斷黃金 / 死亡交叉 | 5 日、20 日均線 |
| RSI | 判斷超買（>70）或超賣（<30）區間 | 14 日 |
| MACD | 趨勢動能變化，判斷多空轉折 | 12 / 26 / 9 |
| 成交量變化 | 量能是否異常放大 | 與 5 日均量比較 |

### 3.4 AI 買賣時機分析

#### 觸發條件（條件式觸發，節省費用）

以下任一條件成立時，才呼叫 AI API 進行分析：

- RSI 跌破 30（超賣訊號）或突破 70（超買訊號）
- MA5 上穿 MA20（黃金交叉）或下穿 MA20（死亡交叉）
- MACD 柱狀圖由負轉正（或由正轉負）
- 成交量超過 5 日均量的 2 倍
- 使用者手動按下「立即分析」按鈕

#### AI 輸出格式

```
建議方向：買進 / 觀望 / 賣出
信心程度：高 / 中 / 低
理由說明：（自然語言，例如）
  RSI 從超賣區（28）回升至 35，顯示賣壓減輕；
  MA5 剛上穿 MA20 形成黃金交叉，短線動能轉強；
  建議可考慮分批買進，停損設於近期低點下方。
風險提示：AI 判斷僅供參考，不構成投資建議。
```

#### 快取機制

- 同一檔股票若技術指標無明顯變化，顯示上次分析結果，不重複呼叫 API
- 介面顯示「上次分析時間」，讓使用者知道資料新鮮度

### 3.5 自訂觸發條件

- 使用者可在設定頁面自行調整觸發條件閾值（例如 RSI 門檻改為 25/75）
- 可選擇開啟 / 關閉特定觸發條件
- 設定儲存於 SQLite

### 3.6 模擬交易系統

- 初始虛擬資金預設 NT$1,000,000，可自行設定
- 模擬買進：輸入股數 → 計算成本（含模擬手續費 0.1425%）
- 模擬賣出：計算損益（含模擬交易稅 0.3%）
- 即時顯示：持倉成本、現值、未實現損益（金額 + %）
- 完整交易記錄（時間、股票、方向、價格、股數、損益）
- 虛擬資金餘額即時更新

### 3.7 績效追蹤

- 累積損益曲線（折線圖）
- 勝率統計（獲利筆數 / 總交易筆數）
- 最大獲利 / 最大虧損交易
- 平均持股天數
- 可依時間區間篩選（本週 / 本月 / 全部）

### 3.8 歷史回測

- 選擇股票代號與回測時間區間（最長 2 年）
- 套用目前的觸發條件設定，模擬歷史期間的 AI 建議
- 計算假設完全依照 AI 建議操作的理論損益
- 與「買入持有」策略比較績效
- 顯示：勝率、最大回撤、年化報酬率

> ⚠️ 回測結果不保證未來表現，僅供策略驗證參考。

### 3.9 Gmail 通知

- 有新的 AI 買賣訊號時自動寄送通知信
- 信件內容：股票名稱、建議方向、關鍵指標數值、AI 理由
- 每檔股票同一訊號每天最多通知一次（避免重複）
- 可在設定頁面開啟 / 關閉通知功能

## 5. AI 引擎設計

### 5.1 可切換架構

AI 引擎採用**策略模式（Strategy Pattern）**設計，所有引擎實作同一個抽象介面，可在設定頁面或 Header 下拉選單即時切換。

```python
# 抽象介面（所有 AI 引擎都要實作）
class AIEngine:
    def analyze(self, symbol: str, indicators: dict) -> AnalysisResult:
        raise NotImplementedError

# 各引擎實作
class ClaudeEngine(AIEngine): ...
class OpenAIEngine(AIEngine): ...
class GeminiEngine(AIEngine): ...
```

### 5.2 支援的 AI 引擎

| 引擎 | 模型 | 費用估算（/月） | 備註 |
|------|------|--------------|------|
| Claude | claude-sonnet-4 | NT$20–80 | 分析品質高，解釋清楚 |
| OpenAI | gpt-4o | NT$30–100 | 品質相當，生態完整 |
| Gemini | gemini-1.5-flash | 幾乎免費 | 免費額度極大 |

### 5.3 切換方式

- Header 下拉選單即時切換，無需重啟系統
- 切換後的分析設定儲存於 SQLite
- 各引擎 API 金鑰分別設定於 `.env`

## 6. 自動下單系統

### 6.1 下單模式

#### 🔔 通知後確認模式（預設）

```
AI 產生訊號
    ↓
發送 Gmail 通知（含建議理由）
    ↓
使用者收到通知 → 手動到介面確認
    ↓
點擊確認 → 執行模擬下單
```

#### 🤖 全自動模式

```
AI 產生訊號
    ↓
自動執行模擬下單
    ↓
發送 Gmail 通知（告知已下單）
```

### 6.2 風險保護機制（模擬模式）

即使是模擬交易，也設定保護機制以養成良好習慣：

- **單次下單上限**：每次最多買進虛擬資金的 20%
- **每日虧損上限**：單日虧損超過虛擬資金 5% 自動停止
- **停損設定**：持倉虧損達設定比例（預設 -8%）自動賣出

### 6.3 擴充預留

未來真實下單擴充時，只需新增券商 API 實作：

```python
class BrokerEngine:
    def place_order(self, symbol, action, quantity): ...

# 預留實作
class FugleBroker(BrokerEngine): ...     # 富果
class MasterlinkBroker(BrokerEngine): ... # 元富
```

## 9. 非功能需求

### 9.1 可擴充性

- 所有交易模式（波段 / 當沖）以策略模式設計，方便新增
- AI 引擎模組化，新增引擎只需實作抽象介面
- 技術指標計算模組獨立，可輕鬆新增指標
- 通知渠道模組化，未來可擴充 LINE 或 Telegram
- 券商下單模組預留抽象層，未來接真實下單不影響其他模組

### 9.2 資料安全

- API 金鑰儲存於 `.env` 環境變數，不寫入程式碼
- `.env` 加入 `.gitignore`，不進入版本控制
- 目前僅模擬交易，無金融資安風險

### 9.3 法律免責

- 本系統僅供個人練習與研究，AI 分析結果不構成投資建議
- 使用者依據系統建議進行真實交易之損失，系統開發者不負責任
